Step 1 · 灌数据

建专家的大脑

路径 A · Chrome 插件

Chrome 商店搜 "YouTube to NotebookLM" 安装
打开专家的频道 → 点插件 → 批量导入整个频道
推荐新手，零代码

路径 B · 命令行批量

# 抓取 URL 列表
yt-dlp "https://youtube.com/@AlexHormozi/videos" \
  --flat-playlist --print url --playlist-end 300

# 批量灌入 NotebookLM
while read url; do
  notebooklm source add -n {id} "$url"
done < urls.txt

适合 10+ 个专家的批量自动化

Step 3 · Core Innovation

Persona Protocol

不是 prompt，是认知操作系统

Role
他是谁？ "Hormozi 的数字分身，冷血商业操盘手。"
Thinking
遇到问题第一反应？ 定价→Value Equation，增长→"More, Better, New"
Tone
怎么说话？ "Just do the math." / "Volume negates luck."
Anti
绝不会说什么？ 绝不会说"先做好产品再想定价"——永远 Offer 先行
Retrieval
搜什么？ 优先搜定价框架、成交率、LTV/CAC

Anti-Pattern 要足够具体 —— 只有这个专家才会拒绝的事

Step 4 · 双层融合

大脑 + 灵魂 = 回答

每次回答走两步：先搜大脑，再加灵魂

# A. 搜大脑
notebooklm ask -n hormozi \
  "How does Hormozi price SaaS?"

# → 返回他视频里说过的具体案例和数字
# B. 加灵魂
Read hormozi_kb.md

# → 加载说话风格、框架名、反模式

大脑提供内容 + 灵魂提供表达

= 不是模仿秀，是真正的数字分身

反幻觉：NotebookLM 检索不到 → 标注 ⚠️ 推演。绝不编造专家没说过的话。
